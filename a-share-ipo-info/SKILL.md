# a-share-ipo-info · A 股 IPO 信息查询

## 触发条件

当用户请求以下任一操作时加载此 skill：
- "查 XX 的 IPO 情况" / "XX 上市进度"
- "XX 辅导备案/申报/上会审核"
- "XX 发行价/中签率/募集资金"
- 任何涉及 A 股 IPO 流程查询的请求

## ⚠️ 数据纪律

- **函数禁止编造结果**：5 个函数均运行无参（无搜索参数），脚本自动模糊匹配企业名
- **字段名是中文**：所有 akshare DataFrame 返回中文列名，不要臆断英文 key
- **三个 EM 函数每次全量拉取**：无分页，略慢（~10s），正常

## IPO 流程五阶段（时间顺序）

```
辅导备案 ──→ 申报队列 ──→ 上会审核 ──→ 注册生效 ──→ 发行上市
  │            │            │            │
  必过此阶段    必过此前两阶段  审核通过才      才有正式代码
  才能申报      才能上会       有代码          才能查④⑤
```

## 查询流程

**唯一入口：** `python3 scripts/search_ipo.py --name "企业名" [--json]`

```bash
# 人类可读
python3 scripts/search_ipo.py --name "宇树科技"

# JSON 格式（Agent 解析用）
python3 scripts/search_ipo.py --name "宇树科技" --json
```

脚本自动：
1. 拉取三个 EM 全量表（辅导/申报/上会）
2. 模糊匹配企业名（标准化去后缀）
3. 有条码 → 自动查 ④⑤
4. 推断 IPO 阶段
5. 输出结构化结果

Agent 拿到结果后按阶段选择输出内容（见下方「分阶段输出策略」）。

## 阶段推断逻辑

| 出现在 | 阶段 | 说明 |
|:--|:--|:--|
| 仅辅导备案 | 辅导备案 | 辅导状态由报告类型+标题推断 |
| 辅导 + 申报 | 申报队列 | — |
| 辅导 + 申报 + 上会 | 上会审核 | 6位正式代码→注册生效；申报表含「注册」→取申报表状态 |
| 全部 + ④⑤有数据 | 已上市/已注册 | 进入发行信息输出模式 |

> **注册状态优先级**：当上会审核表和申报表同时存在，且申报表 `最新状态` 含「注册」时，阶段取申报表状态（比上会表更新），如「已收到注册申请材料」。

## 输出格式

**首行：** `【企业全称　代码　简称】`（有正式代码时附代码和简称）

**名称差异**标注在首行和阶段之间，格式：
`⚠️ 名称差异：名称A（来源表） vs 名称B（来源表）`

**阶段行：** `阶段：XXX`（不含状态后缀，状态在详情中展示）

**详情按阶段裁剪**（脚本已实现，Agent 直接读取脚本输出即可）：

### 辅导备案阶段
- 报告类型 + 辅导机构 + 辅导状态（辅导中（第N期）/ 辅导工作完成 / 辅导验收通过）+ 派出机构 + 备案日期
- 不显示申报信息（后续还没发生）
- 不显示报告标题（期数信息已提取到辅导状态中）

### 申报队列阶段
- 申报信息：最新状态 + 拟上市地点 + 注册地 + 更新日期
- 中介机构：保荐机构 + 律师事务所 + 会计师事务所
- 招股说明书：独立成节，输出 PDF URL
- 不显示辅导信息（辅导完成是申报前提）

### 上会审核 / 注册阶段
- 审核信息：上会日期 + 审核状态 + 上市板块 + 注册地 + 主承销商 + 拟发行 + 拟融资 + 发审委委员
- 上市日期：已上市才显示
- 代码：6位数字→显示「股票代码：XXXXXX　简称」；非6位→「尚未取得正式股票代码」
- 公告日期：不显示（审核结果已出的情况下无参考价值）
- 阶段：申报表含「注册」状态时优先取申报表状态（如「已收到注册申请材料」）

### 已上市阶段
- **基本信息**：上市地（按代码推断：9/8→北交所，688→科创板，3→创业板，0/2→深主板，6→上主板）+ 注册地 + 上市日期
- **中介机构**：保荐机构 + 律师事务所 + 会计师事务所（不重复主承销商）
- **发行信息**：发行价格/数量/面值/市盈率/净资产/中签率/募资净额/各日期
- **IPO 详情**：上市地（缺失时按代码推断）/ 承销方式 / 上市推荐人 / 发行方式 / 股本 / 募资明细
- 不显示审核信息（已上市，审核是历史）

## 已知限制

- 三个 EM 函数无参全量拉取（无服务端搜索），每次查询 ~10s
- 企业名模糊匹配在客户端完成（`str.contains` + 标准化去后缀）
- 辅导备案表企业名含全称，用户简称可能匹配到多条（如"科技"），取第一条
- `stock_ipo_summary_cninfo` / `stock_ipo_info` 仅对已有 6 位正式代码的企业可用
- 申请代码（A 开头如 A26003）不可查询④⑤
- 已封禁的同域函数（不在本 skill 范围）：
  - `stock_ipo_benefit_ths` — API broken (AttributeError)
  - `stock_ipo_ths` — 数据冻结 2026-03
  - `stock_new_ipo_cninfo` — 数据冻结 2023-06
  - `stock_ipo_hk_ths` — 港股
