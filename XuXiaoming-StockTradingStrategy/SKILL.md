### 操作策略输出

> **强制执行清单。Step -1 先确认日期和市场状态。Step 0 验证数据管线五层产出。** 只要触发本 skill，必须按 `references/execution-checklist.md` 顺序执行，不可跳步。
>
> **架构：单 Agent 统稿 + 博客独立比对。** 主 Agent 独立完成全部分析（读数据→四指数推演→跨市场合成→操作建议→总结→极简版），**不拆子 Agent 做七层递进**。唯一例外：第九节博客对照由独立子 Agent 读取博文（A20 三层隔离），主 Agent 只接收摘要后追加比对。**不存在"轻量""仅剩输出""已有分析直接格式化"等快捷模式——每次输出都是完整分析。**
>
- 模板结构：一·市场全景 → 二·主板分析 → 三·科技分析 → 四·跨市场交叉 → 五·操作建议 → 六·关键观察位 → 七·总结 → 八·极简版 → 九·博客对照。
- 每个指数七层递进：年线→趋势→结构→序列→**分钟线（第四层）**→裁决→关键位。分钟线讨论信号强度/置信度，不作为直接操作依据。
- **文件名规范** 🔴：`reports/YYYY-MM-DD_主题_YYYYMMDD_HHMMSS.md`（参见 `references/strategy-output-template.md` v1.9）

**共振分析强制要求**：在第三节「趋势-结构-序列分析推演」的趋势分析中，必须包含五指数共振的历史轨迹（至少追溯最近的共振状态切换节点），用于强化震荡判断。共振数据来自 `verdict_v7.csv` 的 `resonance` 列（六档：强共振_上升/强共振_下跌/偏共振_偏多/偏共振_偏空/混合/分化）。共振从「强共振」塌成「混合」= 趋势解体，是震荡市的最强证据之一。引用原文 [20190731]「多品种共振，指数之间有共振作用」。

### 核心洞察

**方法论不是被市场"改变"的，是被市场"筛选"的。** 两策略轮换系统在2019年已完备——牛市筛选出趋势策略，熊市筛选出结构策略，辅助工具被证明无用后自然废弃。8年演化只是方法论中真正有用的部分被反复验证的过程。

**他用波浪理论重新编码一切外部叙事。** P5的"高波动整理"被编码为"2浪回调"，P6的"科技成长主导"被编码为"3浪上升"。全文"主线""结构性"出现0次——纯指数级技术分析是他的方法论边界。

## 第一层 · 能力层

### 公理（时期不变）

#### 公理1：循环与平衡

- **哲学层面（2种属性）**：市场只有上升和下跌。"震荡市只是形态描述，在哲学上不应该这么思考问题。"（2019-04-29）
- **操作层面（3种状态）**：

```
状态① 单边上升 → 满仓持股，趋势不破不动
状态② 单边下跌 → 空仓防守，等趋势突破
状态③ 震荡/无方向 → 结构策略（高抛低吸/随缘/多看少动）
```

循环：①或② → ③ → ①或② → ③……震荡不是"过渡"，是夹在两次单边之间的独立操作阶段。

#### 公理2：规则永远覆盖预测

预测和规则同时运行，冲突时规则无条件优先。预测权重从约 60% 降至约 15%（行为推断），是不可逆的轨迹。

---

### ⚫ 核心模型

#### 模型1：两策略轮换系统 — 一致性 1.00 | 时期不变

交易体系是一个**市况驱动的两策略轮换系统**。

| | 趋势策略（主） | 结构策略（辅） |
|---|---|---|
| **市况** | 趋势市 | 震荡市 |
| **信号** | 趋势突破/破位 | 结构形成 |
| **方式** | 右侧交易 | 左侧交易 |
| **仓位** | 满仓/空仓 | 六成仓± |
| **金句** | "趋势为王" | "结构修边" |

**切换**：①预判（震荡≥半年+连错≥7次）→ ②布局（降低结构权重）→ ③确认（趋势突破满仓/破位空仓）

**案例**：2024年9月24日趋势突破→徐小明瞬间从"随缘"切换为"不能总是等结构了，要尊重市场"——这正是模型1预设的切换机制被触发，不是方法论突变。

**失效**：趋势突破后再次破位→机械止损。

---

#### 模型2：循环哲学与转换信号 — 一致性 0.80

用"循环+平衡"哲学预判市况转换——不是检测震荡结束，而是**预判**震荡必然转化为单边。

**趋势连错七次**：2019深成指连错7次→第8次突破成功；2020上证独立复现。⚠️ 仅时期一有执行记录。

**转换信号**：震荡≥半年 | 连错≥7次（时期一） | 双通道收敛 | 三角形末端 | 指数协调恢复

---

#### 模型3：工具集持续简化 — 一致性 0.80 | 时期依赖

从 15+ 种收缩至"趋势+结构+序列"三元核心。**2024年划下的线，此后未能跨越回去。**

**"不重要"检测**（时期三核心工具，2026原文定义）：
> "不重要 = 不改变操作结果。过程是怎样波动的都不能导致结果产生。"
- 操作结果 = 趋势突破/破位 + 结构形成 + 钝化消失
- 量化：61.1%的交易日不改变操作结果 → "过程中的交易日不重要"
- 最长连续不重要：42天（近两个月无需任何决策）

**浪型地位变迁**：时期一核心（44–55次/年）→ 2023年宣告"浪型不是规则"→ 时期三仅18次。

**演化逻辑**：牛市→趋势策略好用→全工具集维持。熊市→趋势反复挨打→结构策略上位→辅助工具被证明无用→自然废弃。市场没有改变方法论，只是筛选出了真正有用的部分。

---

#### 模型4："不做"体系 — 一致性 0.72

全年 60–80% 的交易日处于此状态——不是错过，是有纪律的放弃。

**"随缘"**（⚠️ 时期三专属，2024年集中）：
- 精确定义："有结构就有标准/缘分，没有就算了"（2024-06-20）
- 本质是**盈亏比驱动的放弃**：2浪A仅跌7-8%，B浪反弹空间更小→必须精确抄底→没有结构=没有标准=不强行交易
- 生命周期：2019萌芽→2024爆发19次→9/24后退役→2025仅1次→2026归零

**"多看少动"**（全时期，频率随时期变化）：
四种精确触发：①结构形成后等结果 ②"三无"行情 ③信号间隔期（有时间边界）④指数分化

### 基础（9项·三时期不变）

| 维度 | 特征 |
|------|------|
| **语调** | 直接、指导式——不学术、不文艺、不模棱两可 |

| **结构** | "判断→操作"二元——每篇必有明确结论和操作建议 |
| **量化习惯** | 始终给数字——时期一给点位（9742），时期三给时间窗口（"本周上半周之前"） |

### 时期一（2019–2021）：传教式长文

**在基础之上叠加**：

- **句式**：长复句+因果链，10步论证模板（现状→数据→推理→结论→操作→纠错→心态）
- **独有词汇**：哲学词汇（"循环+平衡""新生→鼎盛→衰退"），全套工具术语
- **独有短语**：「趋势为王结构修边」「痛下决心防范风险」「不退」「天时地利人和」「一剑封喉」
- **确定性**：极高——"痛下决心""精准""一根线都不差"
- **节奏**：先建框架→铺数据→推结论→给信心，每篇结尾必有信念加固

### 时期二（2022–2023）：过渡期

**在基础之上叠加**：

- **句式**：规则句替代因果链，5步论证（规则→判断→结论→纠错→心态）
- **独有词汇**：「预测」几乎消失（2023年仅1次），基钦周期替代天时地利人和，「多看少动」爆发（30+次/年）
- **独有短语**：「按规则踏踏实实的做就行了」「多看少动」「浪型不是规则」「交易定力」
- **确定性**：降低——"不犯原则性错误"替代"精准"
- **节奏**：开门见山→给结论→不铺陈

### 时期三（2024–2026）：极简执行

**在基础之上叠加**：

- **句式**：极端短句，2步论证（判断→结论）。2024年平均641字/篇，42%不足500字
- **独有词汇**：工具术语全部归零，心态词汇替代方法论词汇，「随缘」2024爆发19次后退役，「化繁为简」从不言到言，「耐心」136次/年
- **独有短语**：「随缘」「化繁为简」「趋势之上持股，破趋势减仓」「谨慎乐观」
- **确定性**：拥抱不确定——"随缘""等""耐心"。分析可以复杂，操作必须简单
- **节奏**：午评式快节奏（每日1–2篇），不再论证"为什么"，直接给"是什么"

### 表达滞后

徐小明的实际操作领先方法论陈述 2–5 年。方法论变化先出现在操作中，后出现在话语中。

### 铁律（不可违反）

```
趋势为王 > 结构修边 > 序列辅助
```

> [20190210] 如果突破趋势，你就甭管结构了。**趋势为王，结构修边。**
>
> [20190804] 但**序列并不是交易标准**……序列不是交易规则。
>
> [20190812] 买入的原则要符合交易的原则，要么突破趋势，要么底部结构形成。**序列不是买入原则，结构和趋势才是。**

三层优先级不可颠倒。冲突时上层覆盖下层。分钟线不独立裁决——分钟线检测结构信号，日线做最后确认。

**铁律推论：市况决定结构含义。** 同一结构信号（顶/底结构），在不同市况下操作含义完全不同：

| 市况 | 顶结构 | 底结构 |
|------|------|------|
| **趋势上行/偏多** | 结构修边——真的有压制作用。首次→警戒，二次→减仓（R3.1/R3.2） | 趋势为王——底结构不抄底，等趋势破位确认（R3.3） |
| **趋势下行/偏空** | 趋势向下——顶结构冗余，趋势本身就说空 | 等趋势突破——底结构是入场前提，但需BS筛选通过（R3.3） |
| **震荡** | **取决于仓位。** 已入场→按趋势规则修边（R3.1/R3.2）。观望中→不操作（本来就没仓位） | 入场信号——震荡中唯一符合买入原则的触发点。底结构+BS筛选通过→试探。不是"更准"，是"唯一有标准的路"。没底结构就随缘。 **v4.5.6 方向判别**：上行侧→看出口方向（偏多=看多76%，偏空=观望65%）；下行侧→看钝化次数（BS≥2=试探100%，BS=1=陷阱38%，BS=0=抛硬币）。 |

> 震荡市最典型的特征：顶结构频繁（反弹到上沿自然触发），底结构稀缺（下沿未到或到了没钝化）。**不要数顶结构次数制造恐慌——要看底结构为什么不出。** 底结构不出，震荡就不会结束。徐小明：「震荡趋势挨打，震荡的过程中高低点又没有结构。」（2019-10-31）

### 全流程框架：「夹在两次单边之间」

> 徐小明：「震荡不是'过渡'，是夹在两次单边之间的独立操作阶段。」
>
> 这是整个交易体系的全流程框架——与铁律并列。铁律管**信号层的优先级**（趋势 > 结构 > 序列），全流程框架管**阶段层的转换**（从哪来 → 震荡 → 往哪去）。

```
前一段单边（来路）         震荡               后一段单边（去路）
─────────────           ────               ─────────────
上行 → 已减仓/空仓     观望/等底结构         向上突破 → 追买
                                            向下跌破 → 清仓

下行 → 底结构试探入场   持股/顶结构修边      向上突破 → 加满
                                            向下跌破 → 止损
```

**三层含义：**

| 层 | 内容 | 依据 |
|------|------|------|
| **来路决定仓位** | 从上行转来 → 仓位已低（顶结构早减了）。从下行转来 → 已有仓位（底结构已入场）。仓位不由震荡中的结构决定——由震荡的来路决定 | 2019-2026 全量回测 |
| **仓位决定看什么** | 仓位低 → 看底结构（找入场点），顶结构不用管（本来就没仓位）。有仓位 → 顶结构修边、底结构加仓 | R3.1/R3.2/R3.4/R3.5 |
| **去路决定最终操作** | 趋势向上突破 → 买/加。趋势向下跌破 → 清/止损。震荡中的结构只是操作点，不是方向点——方向由趋势改变一次性决定 | R1 |

**回测支撑（2019-2026，117段震荡）：**

| 来路 | 反向结构 | 性质 | 转涨率 | 操作 |
|------|:--:|------|:--:|------|
| 上行 | 有/无顶 | 趋势中继 | **72%** ↑ | 观望/持股——趋势为王 |
| 下行 | 无底 | 下跌中继 | **32%** ↑ | 不入场——"高低点又没有结构" |
| 下行 | 有底+持续 | 摸底 | **67%** ↑ (+3.0%) | 试探——"有结构就有标准" |

> 框架准确率 **69%** vs 随机 50%。长周期（60日）验证：上行来 +1.2%（胜率61%），下行有底 +4.9%（胜率100%）。CHOP轨迹规律：收敛=死亡信号，V形波动=健康。详见 `references/oscillation-origin-framework.md`。

**CHOP 角色**：趋势可信度计，非趋势强度计。CHOP 越高趋势越不可靠 → 结构权重越大。震荡+chaotic 是「钝化反复→结构级别加大」的密集区（类924前夕微观版）。

**非对称滞回设计（v4.6.3，回测验证）**：

| 方向 | 假信号率 | 滞回效果 | 引擎策略 |
|------|:--:|------|------|
| ↑ 上穿 61.8 | 36.7%（18/49天为单日尖刺） | **滞回拖后腿**（延迟18天多亏+16.10%，Sharpe 0.31→0.22） | **单日即切**，无滞回 |
| ↓ 下穿 38.2 | 94%（触及即弹回） | **滞回必须**（持续≥3天：胜率70%+2.1% vs 1天：58%+1.1%） | **≥3天确认** |

> 两条线不对称的原因：上穿 61.8 是真正的恐慌/分歧爆发，即使次日回落当天也真跌；下穿 38.2 是短暂安静然后继续乱。不是 bug——是 CHOP 在两端的数据性质不同。

### 规则 6：分钟线结构修边

> **L3 分钟线层**：分钟线检测 → 日线确认 → 裁决升级。分钟线不独立裁决、不推翻日线方向——只升级不降级。
>
> [20190106] 这相当于我**把大周期定到周五，把小周期定到周五的第一个小时**。
>
> [20190807] 以往都是**小周期先形成钝化再到大周期**。

**检测工具**：60/90/120分钟 MACD(4,30,4) 钝化→结构形成，三周期共振等级 S/A/B/C。

| 共振等级 | 条件 | 含义 |
|:--:|------|------|
| **S级** | 三周期同时形成结构 | 最强信号，需日线确认 |
| A级 | 双周期同时形成 | 强信号 |
| B级 | 单周期形成 | 参考 |
| C级 | 仅钝化未形成结构 | 关注 |

#### M1：分钟线底结构修正

```
分钟线检测到 S/A 级底结构（≥2周期共振）
  → 市况判断：
      趋势市/偏趋势/震荡+chaotic → 允许触发
      震荡+fuzzy → 禁用（噪音，17次胜率29%）
  → 日线确认（至少满足其一）：
      ① 日线低9（当天或未来1-2天）
      ② 日线底部结构（bs=1）
      ③ 强共振下跌（恐慌出清）
      ④ dd20 < -5%（急跌）
  → 日线裁决=观望 → 升级为试探
```

#### M2：分钟线顶结构确认

```
分钟线检测到 S/A 级顶结构 + 日线持股(警戒)
  → 确认顶结构级别，建议减仓（不自动修改裁决）
```

#### M3：分钟线金叉共振

```
三周期金叉共振 + 震荡 + CHOP 收敛（5日内下降≥3）
  → 关注趋势突破，观望→观望(偏多)
```

**市况过滤**（来自 v4.4.1 回测）：

|------|:--:|------|
| 趋势市/偏趋势 | 胜率 67%，均收益 +1.7% | 允许 |
| 震荡+chaotic | 钝化反复→结构级别加大 | 允许 |
| **震荡+fuzzy** | **胜率 29%，均收益 -1.1%** | **禁用** |

> [20191031] 震荡趋势挨打……震荡的过程中高低点又没有结构。
>
> [20230614] 结构也是按收盘价，只不过分钟线的结构，是按这个分钟周期的收盘价……**就是按周期收盘来定结构的。**

> [20230608] 一天交易时间是240分钟……我比较常用90分钟，60分钟以下的结构级别太小，不具备可操作性。

| **R8.7** | 叠加低位：dd60<-5% + NH/NL 弱势 + 裁决非多头 | **方向锁定。** 裁决中位 1 天转正，历史 11/11 最终双向确认（dd60 最低均 ≤-7%）。大盘低点叠加个股低点 → 裁决敢于在低位试探。**阈值分层（v4.6.6 收紧）：** R8.7 监测(-5%) → R8.10 底部连错(-5%，同线) → H11 磨底(-8%) → H11 恐慌(-15%)。R8.7 与 R8.10 统一用 -5%——两个规则在找同一个东西（底部伏笔），不该用不同门槛。浅跌区 -3%~-5% 的 4 段 NH/NL 未确认案例已被此收紧排除。 | 15 段全量，11 段双向确认（均在 -7% 以下） |

| **R8.11** | NH/NL 确认时态 | **第一次确认才有效。** \"叠加低位\" 的有效期 = NH/NL 首次确认前后 2 周。确认已持续数周→不是\"个股低点\"→失去叠加意义。2024年9月 NH/NL 从分化延续→首次确认=拐点（924暴涨）。2025年5月 NH/NL 已确认 3 周=惯性（连错后仅震荡） | 2024 vs 2025 对比 |

**叠加低位核心逻辑 + 确认新鲜度判别**：呼应徐小明"大盘低点叠加个股低点，安全系数再次提高"——

> "叠加低位"不是所有低位都有效——只有 NH/NL 的确认是**新鲜的**（首次确认前后 2 周内），才构成真正的"个股低点"。陈旧的确认（已持续数周）说明个股已经不在低位，叠加窗口已关闭。

判别法则：
```
大盘低位 (dd60<-5%) + NH/NL弱势 (分化延续/预备)
              │
        裁决试探/持股
              │
    ┌─────────┴─────────┐
    │                   │
NH/NL 2周内首次确认    NH/NL早就确认了
    │                   │
    ▼                   ▼
  伏笔                 噪音
 连错=探底             连错=摩擦
 后续大涨              后续震荡
```

- 裁决管买卖 → 低位时裁决先行试探（中位 1 天）
- NH/NL 管安全 → 后验确认环境（滞后 3-51 天）
- 两条线独立，低位时裁决不等 NH/NL
- 如果 NH/NL 反过来先动了 → 轻仓试，等裁决确认后加仓
- **叠加窗口有效期 = NH/NL 首次确认前后 2 周。过期作废。**

**裁决连错模式**（7 年裁决数据）：连错是常态（19 个集群，最多 5 次）。底部连错（dd60 深负）≠ 永久失败——历史上无真正双重失败（裁决退回 + NH/NL 永不确认）。2024 年 7-8 月 4 次连错 → 第 5 次 9/11 成功（924 前 13 天埋伏），NH/NL 最终 9/27 确认。

>
> [20260710] 个股在最近的四个月，其实是非常弱的……分化也有平衡属性，个股有弱于大盘的时候就有强于大盘的时候……大盘低点叠加个股低点，安全系数再次提高，可操作性就很强了。

### 数据中间层 · 独立数据抽取与处理（v4.4.2 新增）

> **data_layer/** 是 skill 自带的数据中间层，使 skill 在任何环境下都能自行拉取数据、处理生成 CSV，不依赖用户本地的行情中间层。
> 日线：akshare 优先 + 时效检测。成功→检查最新日期是否滞后→滞后则 Sina 补缺失交易日。失败→Sina fallback。带磁盘缓存。

```
data_layer/
├── build.py       # 一键构建：拉取 → 处理 → 输出全部 CSV
├── fetch.py       # 数据抽取：从 akshare 拉取五指数全量日线 OHLC
└── process.py     # 数据处理：MACD结构 + 九转序列 + 裁决引擎 + 分钟线
```

**使用方式：**

```bash
# 仅拉取原始数据（需联网）
python data_layer/fetch.py

# 仅处理已有数据
python data_layer/process.py

# 一键构建（推荐）
python data_layer/build.py
```

**数据流：**

```
akshare API
    ├── fetch.py --mode daily
    │   └── data/daily_raw.csv          ← 五指数日线 OHLC (2018年起)
    ├── fetch.py --mode minute
    │   ├── data/minute_raw_60_*.csv     ← 60分钟线 (120天回溯)
    │   ├── data/minute_raw_120_*.csv    ← 60→120重采样
    │   ├── data/minute_raw_30_*.csv     ← 30分钟线 (60天回溯)
    │   └── data/minute_raw_90_*.csv     ← 30→90重采样
    │
    ↓ process.py
data/structure_signals.csv  ← MACD(4,30,4) 钝化→结构信号
data/turn_sequence_events.csv ← 九转序列事件(日/周/月)
    ↓ verdict_v7.py
data/verdict_v7.csv         ← 日线裁决(含多指数共振)
    │    ↓ minute_structure_v2.py
    data/minute_structure_v2_{sh,sz,cyb,kc}.csv ← 分钟线结构(四指数全量，需分钟线数据)
    ↓ verdict_v441.py
裁决输出                     ← v4.4.1 分钟线修边
```

**分钟线数据说明**：分钟线结构修边需要 60/90/120 分钟 OHLC 数据，由 `data_layer/fetch.py --mode minute` 自动拉取。`minute_structure_v2.py` 现已覆盖上证、深证、创业板、科创50 四指数（v4.6.6 修复了 INDEX_MAP 扩容后 main() 循环未同步的 bug，以及 realtime.py check_minute_structure 同步遗漏）。

**日线数据源**（v4.6.7）：`fetch_index_daily()` 主源 akshare（`stock_zh_index_daily`）。成功后检查时效——数据滞后则用 Sina 实时接口（`ak.stock_zh_index_spot_sina`）补缺失交易日；异常时全量 fallback 到 Sina。带本地缓存 `daily_spot_cache.json`。

**LLM比对局限性**：市况维度对齐率仅51%，根因是 LLM 把徐小明的"乐观描述"当方向判断、把"低波动"当震荡。引擎的 MA 分类器在时期级别（12/12关键日期）准确，但不适合每日与 LLM 比对。**市况维度不做独立打分。**

> 详见 `references/blind-test-methodology.md` 和 `references/llm-extraction-method.md`

### 盘中实时裁决（v4.6.6）

> 手动触发，仅盘中启用。**核心原则：前日完整报告是盘中分析的唯一基础——没有完整报告就没有盘中简报。** 简报不是"看一下盘面"的独立产品——它的第二节必须引述前日报告的四条核心判断，然后拿今日实时数据逐条对照。如果引述的是一份残版报告（缺A19/共振轨迹/分钟线），对照就是沙上筑塔。

**触发与流程**：

```
用户说「看一下盘面」
  │
  ├── Step -1: market_status() 时间校验（第一个动作，不可跳过）
  │     ├── is_trading_day()=False → 默认生成前一交易日完整报告（盘后全流程）。用户已指定日期则用指定日期，未指定则取 `get_previous_trading_day()`。
  │     ├── 盘前 → 标注「盘前，数据为昨日收盘」，可继续
  │     ├── 开盘/午休 → 正常执行
  │     ├── 收盘 → **自动切换为盘后全流程**：两路并行+合成+博客后置→输出当日完整报告（九节）。不走盘中简报。
  │
  ├── Step 0: ensure_previous_report_exists()
  │     ├── 有 → 返回前日完整报告路径，直接复用
  │     └── 无 → 补跑全流程（不是只跑 verdict_v7.py）：
  │           ① delegate_task 两路并行（主板/科技 Agent，七层递进）
  │           ② 主 Agent 跨市场合成（一~八节，含 A19+共振轨迹+NH/NL）
  │           ③ 独立博客 Agent 读取（A20 隔离）
  │           ④ 主 Agent 追加比对 → 输出带时间戳的 MD 文件
  │
  ├── Step 1: 数据采集子 Agent
  │     ├── data_layer/realtime.py --json         ← Sina 实时行情（六指数）
  │     │   → get_spot() + estimate_chop_14() + estimate_regime() 
  │     │   + check_minute_structure() + get_yesterday_verdict()
  │     └── 返回结构化 JSON（spot/chop/regime/minute/yesterday）
  │
  └── Step 2: 分析子 Agent（独立上下文，与 Step 1 分离）
        ├── 读取前日完整报告（Step 0 返回的路径）
        ├── 提取四条核心判断（直接从报告第二节/五节/七节中抽取）
        ├── 读取 Step 1 的 JSON 数据
        ├── 逐条对照：昨日说XX→今天实际情况XX→判断是否偏离/触发底线
        └── 输出七段盘中简报（~1000 字）→ 推送微信
```

**为什么数据采集和分析必须分离：**
- 数据采集是机械任务（算推分离原则）→ `realtime.py --json` 搞定
- 分析是逻辑任务（对照前日报告的叙事链条，判断今天是否偏离）→ 需要独立子 Agent 的完整上下文
- 两者共处一个上下文时，数据细节会挤压判断空间——参考今天 11:41 残版报告的核心教训

**前日完整报告要求**（无则补跑时，必须走全流程，不可走捷径）：

| 必须包含 | 内容 |
|------|------|
| 九节完整 | 市场全景→主板→科技→跨市场交叉→操作建议→关键位→总结→极简版→博客对照 |
| A19 分列 | 年内顶底结构按市况分列（趋势中/震荡中），数据源标注 |
| 共振轨迹 | 4/21上升→5/14共振顶→6/5塌混合的时间线 |
| NH/NL 详述 | CNHL+HL_Ratio+叠加判断 |
| 四指数分钟线 | 每个指数第四层嵌入，标注信号等级 + M1 禁用/不禁用 |

**盘中简报结构**（七段，~1000 字）：

| 段 | 内容 | 说明 |
|------|------|------|
| 一 | 实时行情快照 | 六指数点位+涨跌幅 |
| 二 | **接续昨日核心判断** | 从前日报告中提取四条判断，每条标注出处（"昨日报告第X节"） |
| 三 | 今日盘中对照 | 逐条对照四条判断：昨日预期 vs 今日实际 → 偏离/符合 |
| 四 | CHOP 盘中估算 | 四指数 CHOP 估算（标注⚠️），科创50 强制三级输出 |
| 五 | 方向估计 | 20 日动量方向 + 是否触发临界条件 |
| 六 | 分钟线快照 | 四指数当日分钟线状态，与前日报告中的分钟线判断对照 |
| 七 | 裁决判断 + 下一步关注 | 基于对照结果更新判断 + 列出今日剩余交易时段需盯的底线 |

**盘中简报第二节格式**（必须引述前日判断，不是列数据）：
```
## 二、接续昨日（{date}）核心判断

引用自《{date} 市场分析与操作策略》（{report_path}）

① **主板「等底结构」。** 昨日报告第X节：深证来路上行+低9→观望偏多，等日线底结构形成→试探。
② **科创50「手放在减仓按钮上」。** 昨日报告第Y节：持股警戒，CHOP=57.1距硬切换4.7点+分钟线S级顶共振。
③ **分钟线集体禁用但科创50不禁用。** 昨日报告第Z节：M1禁用上证/深证/创业板底结构，科创50顶结构有效——是当天最强空头信号。
④ **底结构极度匮乏 + 叠加低位临近。** 上证深证半年零底结构，R8.7 叠加低位距触发差2%。

**今日盘中对照：**
- 科创50 CHOP：昨日57.1 → 今日盘中估算 XX.X | 距61.8 剩 X.X 点 | 触发/未触发底线
- 深证低9：昨日低9第1天 → 今日第2天 | 是否出现底部钝化
```

**设计决策**：

| 决策 | 原因 |
|------|------|
| 前日报告检测用 `ensure_previous_report_exists()` | CSV 就绪 ≠ 报告就绪。没有完整报告就没有四条核心判断可引述 |
| 补跑全流程 ≠ 只跑 verdict_v7.py | CSV 只是数据，不能代替七层递进+跨市场合成 |
| 数据采集与分析分离（两个独立子 Agent） | 机械任务（realtime.py）不侵占逻辑判断的上下文空间 |
| 第二节强制引述前日报告 | 盘中简报的历史锚定——没有昨日叙事链，今日数据只是随机的价格波动 |
| 手动触发，无 cron | 盘中判断需要人工决策，不能自动推送 |


### 博客对照功能 · v4.4.4 新增

> **不干扰分析，仅在输出末尾独立比对。核心判断用 LLM 理解，不用正则提取。**

`scripts/blog_monitor.py` — 给定分析日期，自动抓取徐小明针对该日的操作策略文章（标题+链接+**全文正文**）。

**日期映射（v4.6.7 修正）**：盘后报告用 D 日收盘数据为 D+1 提供操作指引，应对照 **D 日收盘后发布**的博客（标题为 D+1 的周X策略）。周一 D 日→找周一收盘后发布的《周二操作策略》，非周日发布的《周一操作策略》。三重校验（标题绑定+发布窗口+过期过滤）已同步修正。

| 徐小明行为 | 对应关系 |
|-----------|---------|
| T 日收盘后发布"[周X]操作策略" | 数据基于 T 日收盘，策略针对 T+1 |
| 分析日 D | 找发布日期 = D 的前一交易日、标题包含 D 的星期几的文章 |

示例：分析 7/13（周一）→ 找 7/13（周一）收盘后发布的"周二操作策略"（策略针对 7/14 周二）。

**三重校验防止张冠李戴：**

| # | 校验 | 作用 | 示例 |
|---|------|------|------|
| 1 | **标题绑定** | 标题必须匹配 D+1 日的星期几 | 分析周一→只匹配"周二操作策略"，不会错误匹配"周一操作策略" |
| 2 | **发布窗口** | 周一~周四：发布日期=分析日当天；周五：发布日期=周日 | 周一分析→找周一收盘后发布的文章；周五分析→找周日发布的文章 |
| 3 | **过期过滤** | 发布日期太旧的跳过 | 不会用上周的"周二操作策略"去匹配本周一的周二策略 |

**正文提取**：自动解析新浪博客页面的 `articalContent` 区域，提取 `<font>` 标签内的全文（约300-500字）。

**使用方式：**
```bash
# 完整模式：标题 + 链接 + 全文正文
python scripts/blog_monitor.py --date 2026-07-10

# 轻量模式：仅标题 + 链接
python scripts/blog_monitor.py --date 2026-07-10 --no-content
```

输出格式：
```
TITLE: 徐小明：周五操作策略
URL: https://blog.sina.com.cn/s/blog_4d89b834010302de.html
PUBLISH: 2026-07-09
TARGET: 2026-07-10
---BEGIN CONTENT---
[全文正文]
---END CONTENT---
```

**集成流程：**
1. 模型独立完成前六段分析（市况→年线→推演→裁决→观察位→总结）
2. 分析末尾独立运行 `blog_monitor.py --date <分析日>`
3. **用 LLM 理解正文，提取核心判断**（看多/看空/观望、关键依据、关键点位）——**严禁正则提取**
4. 与模型裁决比对，按模板第七节格式输出

正则提取已被证明不可靠：v9 盲测一致率仅 35%，LLM 逐篇提取升至 55-65%。否定句式（"不是不能买"）、历史描述（"昨天我们说过"）、假设条件（"如果跌破则..."）正则全会误判。

若无对应文章 → 静默跳过，标注「当日无操作策略文章」。

**无文章 ≠ 错误**：徐小明在周五收盘后才发布下周一策略，若分析周一且周五文章未发布，返回空是正确行为。

### 裁决引擎 v4（保留参考）

`scripts/verdict_engine.py` — **四引擎合一**。L1月周九转(背景) → L2三合一市况(ADX+ER30+MOM/250天分位/无滞回/偏多=趋势) → L3结构信号(MACD) → 裁决。

**核心裁决规则**（v2，原文一致率70%）：
```python
is_bullish = regime in ('上行趋势', '偏多')
is_bearish = regime in ('下行趋势', '偏空')

if is_bullish:
    if ts: verdict='持股'  # 顶部结构形成(趋势未破)—仅备注
    else: verdict='持股'    # 趋势向上
elif is_bearish:
    if bs: verdict='观望'   # 底部结构+下行趋势(待趋势转)
    else: verdict='空仓'    # 趋势向下
else:  # 震荡
    if bs: verdict='试探'   # 底部结构+震荡
    else: verdict='观望'    # 震荡市
```

**关键设计决策**：
- 偏多=趋势→持股（不等≥7天确认）。原文一致率从41%→70%
- 顶部结构在趋势中不改动作（趋势为王，结构修边）
- 不抄底（下行趋势+底部结构=观望，等趋势转）
- 月周九转标记为备注（`month_seq`/`week_seq`字段），待前向窗口扩展

产出 `data/verdict_daily.csv`（1573天，2020-01-10起）。字段：date/close/regime/trend_confirmed/bottom_structure/top_structure/month_seq/week_seq/verdict/reason。

### 九转序列引擎（v3，2026-07-11）

基于 TD Sequential，徐小明用法：收盘价 vs 4根前收盘价，连续9次同向 = 高9/低9。相等时断裂。8-10均为有效序列值。

**覆盖**：日/周/月三周期六指数，日线额外检测 钝化+9 增强信号。

| 周期 | 高9段/指数 | 低9段/指数 | 年均频率 |
|------|----------|----------|---------|
| 日线 | 31-41 | 26-35 | ~4-5次/年 |
| 周线 | 4-10 | 5-9 | ~1次/年 |
| 月线 | 2-3 | 0-2 | 共2-3段/8年 |

**钝化+9**（同周期钝化状态 + 序列9重叠）：日线仅 2-7 次/指数/8年，极稀少。

**月线低9极端稀有**（9年×5指数仅两段簇）：
- 2018-09~2019-01：五指数全触发（5个月连续，对应2440大底）
- 2023-11~2024-03：四指数触发，上证缺席（对应2635底，级别<2440）
- 2020-2022 和 2025-至今：零触发
- 对比：日线低9 ~15次/年/指数，周线低9 ~4次/年/指数 → 月线低9 约 0.4次/年/指数且成簇出现
- 40天窗口升档器权重基于此稀有性设定

**共振输出**：按日期+方向聚合，输出多指数（n≥2）或多周期（n≥2）共振事件，124次共振（去重后）。

#### ═══ 交叉分析原则（防单指标陷阱）═══

> 序列出信号 ≠ 可以操作。趋势/结构/序列三者不可独立使用，必须按优先级交叉验证。

**优先级链**：`趋势 > 结构 > 序列`（徐小明原文："这个排序的意义在于如果这三者之间起冲突的时候，谁要服从谁"）

**数据验证**：
- 序列整体~70%成功率，但30%失败中 **60% 发生在逆趋势场景**
- 高9 + 上升趋势 → 35%失败率；低9 + 下降趋势 → 38%失败率
- 徐小明原文："序列的缺点就是丢趋势，一个趋势性的行情，序列几乎都是不对的"

**三层过滤决策**：
```
① 趋势层 → 趋势方向 vs 序列方向 → 反向则序列降权
② 结构层 → 有钝化/结构形成 → 增强可信度；无则观望
③ 序列层 → 8-10有效值、高分周期优先、多指数共振加分
```

**四场景速查**：

| 趋势 | 结构 | 序列 | 可信度 | 操作 |
|------|------|------|--------|------|
| 震荡 | 有钝化 | 高9 | **高** | 准备减仓，等结构形成 |
| 震荡 | 无 | 高9/低9 | 中 | 随缘，不作为操作依据 |
| 上升趋势 | 无 | 高9 | **低** | 序列让位趋势 |
| 下降趋势 | 底部结构 | 低9 | 中 | 可信反弹，非反转 |

**核心洞见**：序列是震荡市的工具，趋势市里主动退让。"序列属于小聪明，趋势是大智慧"（2020原文）。

> 详见 `references/operations-layer.md` 交叉分析原则章节

### 连错七次——运行时验证结论

徐小明原文："我判断8、9、10这三次有一次走单边市的概率非常大"——7是统计观察，不是精确触发值。

脚本实测（2019-2026，上证指数，MA20/MA60双通道+20天确认窗口）：

- 52次有效趋势信号，最大连错 8 次（2021Q4-2022Q1）
- 连错≥5 出现4次聚类；连错≥7 仅1次
- **实用阈值**：连错≥5 标记"关注"，连错≥7 标记"预警"——对应徐小明"提前1-3周布局"
- 连错信号只告诉你"震荡→单边即将切换"，不告诉方向（2019Q4→上涨，2022Q1→下跌）
