# 裁决引擎 vs 徐小明原文 逐节点对比

> 日期：2026-07-11 | 引擎版本：verdict_engine.py v2

## 对比方法

- 从徐小明原文中提取 18 个关键操作节点（有明确操作声明的日期）
- 裁决引擎输出每日建议（持股/空仓/观望/试探）
- 判定一致规则：
  - 原文"入场/加仓/满仓/持股" + 引擎"持股/试探" = 一致
  - 原文"减仓/空仓" + 引擎"空仓/观望" = 一致
  - 原文"不理会调整" + 引擎"持股" = 一致

## 核心洞察

1. **市况分类器滞后是主要差异源**（924、3731退潮前）。三合一分位法可替代徐小明未公开的趋势标准，但无法完全消除滞后。
2. **原文预测入场不应作为对比基准**（疫情底、2646底）——引擎按规则走反而是正确的。
3. **裁决逻辑v2解决了两大bug**：结构压趋势（3731）→ 趋势未破不卖；偏多=观望 → 偏多=趋势。
4. **趋势工具在转折点全军覆没**——结构（底部钝化→形成）是唯一提前信号。两者互补：结构做转折，趋势做确认。
