# 操作层 · 代码陷阱速查

## numpy np.select 字符串截断

```python
# ❌ 错误：np.select 默认 dtype 长度取最长的 choices 字符串
cond = np.select([...], ["单边上升", "单边下跌"], default="震荡")
final = cond.copy()
final[mask] = "震荡·有信号"  # ← 被截断为 "震荡"！

# ✅ 正确：强制 object dtype
cond = np.select([...], ["单边上升", "单边下跌"], default="震荡").astype(object)
```

原因：numpy 为固定长度字符串分配 dtype（如 `<U4`），赋值更长字符串时被静默截断。

## 通道实现用当前日K线 → 价格永远不突破

编写通道分类器时，使用 `df["high"].rolling(N).max()` 取通道上轨。滚动窗口包含当天 K 线，导致当天最高价不可能超过"包含自己的最高价"——**0次突破**。

**正解**：`df["high"].shift(1).rolling(N).max()` —— 用前一日通道对比当日收盘。

## 单日截面数据做趋势判断 → "只见树木不见森林"

仅用 `iloc[0]` 取当天数据做判断（如 MA20/MA60 是否交叉），不考虑历史上下文。对于复杂判断（市况、趋势方向），单日快照信号可用性极差。

**正解**：至少拉到 250 天历史数据，用滚动窗口或持久化过滤。参见 `references/multi-dimension-principle.md`。

## 正则提取徐小明原文操作指令 → 误匹配率极高

徐小明的文章包含大量否定句式（"不建议追高"≠加仓）、历史描述（"我们1月4日买入"≠当日操作）、假设条件。正则在`操作`维度误匹配率达 65%，`结构`维度误匹配率达 66%。

**正解**：用 LLM 逐篇提取结构化判断。参见 `references/llm-extraction-method.md`。

## ATR / ADX / Donchian 通道 → 不是徐小明的方法

v4 分类器曾塞入三个非徐小明指标：
- **ATR 滚动百分位** — 量化波动率收敛（Bollinger squeeze 那套），不是徐小明
- **ADX** — Wilder 经典指标，徐小明原文无
- **N 日最高/最低通道** — Donchian/Keltner，非徐小明

徐小明原文提到的"收敛"是指**两条趋势线间距收窄**（短期均线向长期均线靠拢），不是 ATR 波动率收敛。他的通道是两条均线本身构成上下轨（max 线为上轨、min 线为下轨）。

**正解**：`channel_classifier_xu.py` (v5) — 只使用 MA20/MA60 通道 + 间距变化收敛发散 + 收盘价突破/破位。不使用任何外来指标。

## 强共振_上升 ≠ 无条件好信号

六指数共振的时间窗口很关键：
- 刚突破时（0-4 天）出现强共振 → 最差（107 样本，20 日 -1.42%，胜率 37.4%）
- 5-14 天后出现 → 中等
- 典型滞后共振（如 2024-10-08，涨了 36% 后才六指数全共振）→ 见顶

"混合"模式（部分指数已突破、部分还在犹豫）往往是更好的进入点——抓到的是扩散早期而非齐刷刷的尾声。参见 `references/resonance-stratification.md`。
