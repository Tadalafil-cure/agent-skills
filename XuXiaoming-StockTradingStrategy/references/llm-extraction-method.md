# LLM 提取方法

> 2026-07-11 盲测 v10 中确立。正则无法处理徐小明语言的精细语义，LLM 提取是可靠替代。

## 问题

正则提取徐小明原文操作指令，在 `操作` 维度误匹配率 65%，`结构` 维度误匹配率 66%。原因：
- 否定句式："不建议追高" 被正则可匹配为 "加仓"
- 历史描述："我们1月4日买入" 不是当日操作
- 假设条件："如果突破趋势则加仓" 不是实际指令

## 方案

1. 将待比对文章分批写入 JSON（每批 20 篇）
2. 用 `delegate_task` 派发子 Agent，每个处理 2 批（40 篇）
3. 子 Agent 读原文 → 输出结构化 JSON：
   ```json
   {"date":"...","不重要":true/false,"市况":"上升"/"下跌"/"震荡","操作":"减仓"/"加仓"/"观望","结构":true/false,"核心判断":"..."}
   ```
4. 收集所有 batch 的 structured.json，与引擎逐项比对

## 注意事项

- 剔除午评文章（信息不足），只用收盘后 "操作策略" 文章
- LLM 可能把 "单日大阳线" 当 "上升趋势"、把 "低波动" 当 "震荡市"——市况维度的 LLM 判断不可完全信任
- 结构信号维度（68%）和操作维度（54%）是更可靠的比对维度

## 正则提取的已知局限（v13 发现，2026-07-12）

在2023-2024比对中发现正则提取的四大局限，尤其时期三（极简表达式）更为严重：

1. **否定语境误判**：`不需要大幅度减仓`、`不建议追涨` → "减仓""追涨"被捕获为操作指令（实际语义相反）
2. **描述性文本误判**：`各路抄底资金` → "抄底"是描述市场，非操作建议
3. **提取率低**：时期三仅33-43%文章可提取操作方向（vs LLM 87-100%）
4. **强度区分失败**：无法区分"建议不变"（中性观望）vs "一定要买"（强买入），全部归为同级别

**结论**：正则可用于快速初筛和趋势明确年份（2020/2022），但不适用于严肃一致性比对。时期三（2024+）优先用LLM提取。
