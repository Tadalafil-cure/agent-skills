# H11/H12 退震未确认期间应抑制

> v4.6.24 | 2026-07-20 实战教训
> 
> **核心规则**：退震未确认（天数 < 分流阈值）期间，H11（恐慌底试探）/H12（CHOP轨迹升档）/H13（磨底试探）须抑制——退震方向未定，结构信号可能是下跌中继而非真底。

## 触发条件（此规则）

- 震荡来路已判定
- CHOP < 38.2 + regime 翻偏空/下行趋势 → 退震信号触发（Day 1）
- 退震确认天数未满：上行来路需 ≥3d / 下行来路需 ≥4d
- **在此期间，引擎不应输出 试探/持股 裁决**

## 7/17→7/20 案例

| 日期 | 事件 | 深证裁决 | 问题 |
|------|------|------|------|
| 7/14 | 震荡+fuzzy, 五指数混合(0多0空5震荡) | 观望(偏多) | — |
| 7/16 | 震荡, 三周期分钟线死叉 | 观望(偏多) | — |
| **7/17** | **CHOP=35.5 < 38 + regime翻偏空 → 退震Day1** | **持股(恐慌底)+月低9** | ⚠️ H11触发在退震Day1，退震未确认 |
| 7/20 | CHOP=33.4, 继续跌-0.70%, 钝化全消失 | **空仓(关注)** | H11信号被否定 |

**抢跑原因**：引擎检测到 CHOP<40 + 前高CHOP + dd60>15% + 月低9 → 按H11/H12合并逻辑输出 持股(恐慌底)。但此时退震仅Day1（需≥3d确认），退震方向未定——7/17的暴跌可能是真正恐慌底，也可能只是下跌中继的第一天。

**7/20 裁决翻车**验证了后者：7/17不是恐慌底，是下跌中继。

## 为什么退震Day1的H11不可靠

1. **退震方向未定**：Day1的CHOP<38+偏空可能只是单日尖刺，次日可能弹回（见 CHOP 单日跌破38.2的94%假收敛历史数据）
2. **钝化是过程不是单点**：7/17出现三周期底钝化，但7/20就全消失了——"钝化出现→消失"不等于"结构形成→确认"，而等于"多头失败"
3. **恐慌底需要确认**：恐慌底模式隐含"钝化→结构形成→单日大阳"三步。退震Day1只有钝化（第一步），结构未形成、大阳未出现
4. **方向概率不偏向多头**：下行来路退震方向向下概率55%（近随机），不可预判。退震Day1就在向下方向触发试探，本质是赌方向——而框架明确说"不可预判方向只能等退震确认后跟"

## 修复方案

**引擎层**（verdict_v7.py）：在 试探/持股 输出前增加退震确认检查：

```python
# 退震未确认 → 抑制H11/H12/H13试探
if is_exiting_oscillation and exit_days < exit_confirm_days:
    # 退震方向未定，不输出试探
    verdict = "观望"  # 不输出 持股(恐慌底)/持股(磨底)/试探
    reason += " | 退震未确认(需≥{}d,当前{}d)→抑制试探".format(exit_confirm_days, exit_days)
```

**报告层**（输出模板）：退震未确认期间的报告应明确标注"退震未确认，当前方向不可预判，等退震确认后再评估结构信号"。

## 与现有规则的关系

- 此规则是 **H17a（退震框架）对 H11/H12/H13（试探信号）的优先约束**。注意：H17c（下行来路BS≥2→试探）已降级为推论层 S4，不作为规则层约束。
- 优先级：退震确认 > 试探信号（在退震期间，方向判断优先于结构判断）
- 退震确认后：H11/H12/H13 按原规则正常评估
- 此规则也符合铁律「趋势为王 > 结构修边」——退震方向判断属于趋势层，H11/H12属于结构层
