# H17b 回测：上行来路震荡中底结构抑制

> ⛔ **v4.6.24 更新**：震荡中结构信号一律不操作（规则层），BS≥2→试探已降级为推论层 S4。本文记录 v4.6.12 时期的来路区分回测，供历史参考。

## 背景

v4.6.12 裁决引擎的震荡市底结构分支（BS+filter）不区分来路。无论来路=上行/下行/不明，bs=1+bs_ok=True 一律触发 bs_pattern() → 试探/持股。

2026-07-14 实战暴露此缺口：深证来路=上行，bs=1+bs_ok=True，引擎输出"持股(磨底)+月低9"。但徐小明当天博客只等明天 3985 确认，没有任何操作建议。对照框架后发现——上行来的震荡，底结构不应触发操作。

## 修复（v4.6.13）

`verdict_v7.py` 震荡段 bs 分支改为三路：

```
osc_origin == 上行 → 观望 + "来路上行+底结构→关注(不触发)"
osc_origin == 下行 → 原逻辑（BS≥2→试探/持股）
其他               → 观望 + "来路不明+底结构→观望"
```

## 逻辑链

上行来的震荡：
  上行趋势 → 顶结构 → 已减仓 → 仓位已低
  → 震荡中底结构 = 反弹不是反转
  → 不抄底，等出口方向或趋势突破

下行来的震荡：
  下行趋势 → 空仓 → 等入场
  → 震荡中底结构 = 唯一有标准的入场点
  → BS≥2 → 试探
