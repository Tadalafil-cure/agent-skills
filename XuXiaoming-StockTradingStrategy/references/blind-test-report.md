# 盲测验证报告

**日期**: 2026-07-11  
**方法**: 交易日锚定 + 时间对齐盲测（只用当天及之前数据）  
**样本**: 随机300交易日 → 182天有操作策略文章 → 66天引擎判"重要" + 116天判"不重要"

## 方法论（v9）

1. 随机抽 300 个交易日（2019-2026）
2. 对每个交易日，引擎出判断（仅用当天及之前数据）
3. 找到当天收盘后操作策略文章（操作策略，非午评）
4. 展示引擎输出与原文关键句，供人工比对

**核心改进**：以交易日为锚（非文章），直接比对而非正则提取

## 比对维度

| 维度 | 原文关键词 | 引擎对应字段 |
|------|----------|------------|
| "不重要" | 今天不重要/过程中/不改变操作结果 | `result_change == 0` |
| 操作指令 | 减仓/加仓/建议买入/减仓防守 | 引擎信号 + action |
| 结构信号 | 顶部结构/底部结构/钝化 | `struct_form` / `div_lost` |
| 趋势信号 | 趋势突破/趋势破位 | `trend_cross` |
| 市况 | 上升趋势/下降趋势/震荡 | `condition` |

## 结果

**"不重要"维度**：116天引擎判"不重要"，抽样20例全部确认（原文确实没有操作变化）。

**"重要"维度**：66天引擎判"重要"，仅12-15例原文有对应操作指令。引擎精确率约20%。

## 根因分析（2026-07-11）

核心问题：引擎裁决阶段只读单行数据，丢失多时间框架上下文。

- 250天 trend_cross=1 中，118天（47%）在震荡·无信号市况触发
- 其中107天是纯趋势交叉（无结构/钝化配合，确定噪音）
- 引擎把"任何技术信号"等同于"操作变化"，但徐小明只在信号达到操作级别时才行动

**三类假阳性**：
1. 原文明确说"没有结构"但引擎检测到了（引擎误判）
2. 有信号但 Xu 说"过程中/不重要/不操作"（上下文缺失）
3. 引擎检测到信号，但原文是回顾/教育/展望内容（文章类型误匹配）

## 修复

规则引擎裁决阶段加载250日上下文窗口，震荡市纯趋势交叉降级为噪音。
预计引擎精确率从35%升至~55%。
