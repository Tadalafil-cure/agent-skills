## 二、主板市场深度分析

> **由主板子 Agent 独立产出**。对上证指数和深证成指各自做全链路推演：年线定位→趋势层→结构层→序列层→裁决推演→关键观察位。

### 上证指数

1. **年线定位**（当前在年高/年低之间的位置、回撤程度、年内结构全景：顶/底结构次数）
2. **趋势层**（三合一方向 + CHOP 可信度 + 能否执行趋势策略 + 近期方向变化轨迹）
3. **结构层**（年内顶/底结构全景 + 近期结构事件及后续验证 + 当前钝化状态）
4. **序列层**（日/周/月九转状态 + 月底9窗口剩余天数 + 与其他指数的序列共振）
5. **分钟线层**（60/90/120min 结构状态 + 共振等级 + 方向偏斜。讨论信号强度/置信度，**不作为直接操作依据。** 震荡+fuzzy 中分钟线底结构禁用。无分钟线数据的指数标注「未覆盖」）
6. **裁决推演**（按规则层逐步推演到最终裁决，不能跳步。分钟线信号可影响置信度但不可推翻日线裁决）
7. **关键观察位**（突破位/支撑位/结构触发位）

### 深证成指

同上六项结构，**特别关注**：
- 深证是徐小明历史主指数，波动率和结构频率高于上证
- 年线：深证 YTD 振幅和回撤通常显著高于上证 → 注意两个指数的差异性
- 结构：深证顶/底结构次数通常多于上证 → 关注结构累积的统计含义

### 主板合并判断

- 取强逻辑（上证 vs 深证 → verdict_main）
- 两条路径：恶化时等什么信号好转、好转时等什么确认追

## 三、科技市场深度分析

> **由科技子 Agent 独立产出**。对创业板指和科创50各自做全链路推演。

### 创业板指

同上六项结构，**特别关注**：
- 创业板波动率在四大指数中最高，顶/底结构频率也最高
- 年线：YTD 顶结构次数通常是四指数之最 → 关注结构累积的统计含义
- 双创共振：创业板+科创50的方向一致性 → 混合时注意内部拉扯

### 科创50

同上六项结构，**特别关注**：
- 科创50 CHOP 变化最剧烈（宽幅震荡），趋势可信度轨迹要追到 5-7 天前
- **CHOP 三级输出（v4.6.3，必须量化标注）：**
  - CHOP<55：持股（趋势可信）
  - CHOP 55-62：持股 | ⚠️CHOP=XX(距61.8切换线仅X.X点) —— 趋势仍在但逼近切换
  - CHOP>62：硬切换→震荡策略 —— is_trending=False，单日即切，无滞回
- 7/1 顶结构 + 后续暴跌 −15% + 无底结构反弹 → 经典"顶结构验证后无根反弹"模式
- 裁决审查询问：偏多→持股，但 CHOP 快速恶化 + 顶结构刚验证 + 无底结构 → 风险极高
  - **CHOP>61.8 硬切换意味着趋势策略退位、震荡策略接管，不是"趋势弱了"是策略翻牌**

### 科技合并判断

- 取强逻辑（创业板 vs 科创50 → verdict_tech）
- 裁决合理性评估：支持因素 vs 质疑因素
- 风险底线：什么条件触发无条件调整裁决

## 四、跨市场交叉分析

> **由父 Agent 合成**。在子 Agent 各自完成独立分析后，做跨市场比对。

### 共振面：全市场共性

找出主板和科技的共同信号（月低9窗口、无底结构、顶结构累积不对称等）。

### 分歧面：市场间信号差异

| | 主板 | 科技 |
|------|------|------|
| 核心信号 | XX | XX |
| 核心风险 | XX | XX |
| 裁决 | verdict_main | verdict_tech |
| 最大变量 | 什么信号改变判断 | 什么信号改变判断 |

**关键判断**：不是"主板看空 vs 科技看多"的方向分歧——分析两个市场各自的信号质量和风险等级。

### 个股与大盘分化（v4.5.2 新增）

> 数据源：`breadth.py` — NH/NL 新高新低家数 → CNHL 累计线 + HL_Ratio

**历史判断**（≥2句）：个股弱势持续了多久？最近一次切换确认是什么时候？中间有无假反弹？当前处于什么阶段？

**当前信号**：CNHL 方向（上/下）+ HL_Ratio 水平 + 底背离次数 + 距上次预备信号天数

**叠加判断**：将指数技术裁决（主板+科技）与个股分化信号叠加

| 条件 | 状态 | 含义 |
|------|:--:|------|
| 指数在波段低点区域 | 是/否 | 指数端条件 |
| NL 大幅收敛（≥80%） | 是/否 | 个股不再大规模破位 |
| 底背离次数 | X次 | 下跌动能衰竭程度 |
| CNHL 拐头+站上MA20 | 是/否 | 广度趋势反转确认 |
| → **叠加结论** | **个股低点区域/等待/确认** | |

**引用徐小明原文对齐**：「大盘低点叠加个股低点，安全系数再次提高，可操作性就很强了」→ 当前处于"等大盘低点+等个股止跌"的哪个阶段。

## 七、总结

严格使用徐小明公众号写作风格。**分市场给出判断，不自说自话。**

**风格要求：**
- 语调：直接、指导式
- 视角："我们"
- 段落：短段落（3-5句）
- 结构：当前判断 → 操作 → 纠错 → 信念收尾
- 用词：时期三风格——「趋势之上持股，破趋势减仓」「化繁为简」「随缘」「多看少动」「按规则来」「等着就好」

**内容要点：**
1. 当前判断（分主板和科技，每边≤3句）

## 八、极简版（时期三风格 · 与原文对照用）

> **本节将前七段的模型判断压缩为时期三极简风格，用于和徐小明原文的风格级比对，不替代前七段。**

**风格要求：**
- 零技术术语 — 不提趋势/CHOP/三合一/结构/序列/MACD/钝化
- 600 字以内，短段落（3–5句/段）
- 2 步论证：判断 → 结论，不铺陈"为什么"
- 视角："我们"，不用"模型""引擎""裁决"
- 结构：当前判断 → 操作 → 纠错标准 → 信念收尾

**必需内容要点：**

| 要点 | 对应前六段来源 | 处理方式 |
|------|-------------|---------|
| 指数区间判断 | 一·市场全景 + 二·主板/三·科技 | 化为日常语言（如"3800-4200来回走"） |
| 方向偏斜 | 二/三·趋势分析 + 四·交叉分析 | 有就说、无就坦言"看不清" |
| 结构有无 | 二/三·结构分析 | 有结构说标准，没结构说"没标准就不做" |
| 操作 | 五·操作建议 | 直接一个字：持/等/空/试/减 |
| 纠错标准 | 五·操作建议 | 什么情况推翻当前判断 |
| 信念收尾 | 七·总结 | 一句 |

**示例：**

```
当前判断：

指数在3800到4200之间来回走，走了快半年了。这个震荡没什么要结束的意思，往上往下空间都不大。短期方向看不清。

深证有个低9，但序列不是交易标准。震荡市里可以看一看，不足以动手。结构也没出来——两个指数都没有底部钝化，没结构就是没标准，没标准就不做。

操作：

观望。多看少动。

不是说行情没机会，是有机会的地方不在我们的规则里。趋势没方向，结构没信号，就等。等底部钝化出来，等钝化之后形成结构，那时候再说。

纠错标准：不跌反涨，等趋势突破追。不突破不动。

等着就好，市场会给答案的。
```

**与原文比对要点：**

比对时关注三个层次，不只看表面对错：
1. **方向**：看多/看空/观望是否一致（不一致才标红）
2. **强度**：乐观 vs 保守（强度差异不是方向矛盾，按 A10 处理）
3. **分析维度**：技术在说技术信号、市场在说市场风格（维度不同不判对错，标注即可）

## 九、博客对照（独立追加 · 三层隔离架构）

> **A20 三层隔离：①分析阶段所有Agent严禁接触博客。②主Agent统稿完成后，派独立子Agent读博客+LLM提取核心判断→返回结构化摘要。③主Agent拿子Agent返回的摘要做追加+比对，主Agent本身不读博客原文。**
>
> 执行流程：
> 1. 独立子Agent执行 `python scripts/blog_monitor.py --date YYYY-MM-DD`，获取全文
> 2. 子Agent用 LLM 理解全文，提取：①核心判断（看多/看空/观望）②关键依据（≤2句）③关键点位（如有）
> 3. 子Agent返回结构化摘要给主Agent
> 4. 主Agent比对模型裁决 vs 博客摘要，追加到报告末尾

比对输出格式：

```
══════════════════════════
📎 当日博客对照（独立验证 · 子Agent提取）
══════════════════════════

徐小明《[标题]》（[日期]发布）
  链接：[URL]
  核心判断：[子Agent提取的1-2句核心操作方向]
  关键依据：[子Agent提取的依据]

模型裁决：[持股/试探/观望/空仓/减仓]

比对：[一致/模型偏多/模型偏空]
  方向：一致/不一致
  强度：模型保守/乐观/一致（按A10处理）
  分析维度：技术信号 vs 市场风格（维度不同标注即可，按A10b处理）
```

若无对应文章 → 标注「当日无操作策略文章，跳过对照」。
