# v5 双模引擎设计文档

## 核心逻辑

**通道状态即市况**——不需要额外20天确认。深证通道说"单边_上升/下跌"就是单边模式，说其他就是震荡模式。

```
单边模式 → 趋势+结构，连错归零不用
震荡模式 → 等结构+等突破，连错可预警
```

## 为什么连错只能在震荡模式

连错追踪的是日线MA20/60趋势交叉连续失败。进入单边市后：
1. 价格持续在通道上方/下方，不再触发新的趋势交叉信号
2. 计数器冻结在上一次震荡期的值
3. 继续用这个冻结值做预警是噪音

**v5修正**：市况切换为"单边"时，`fail_count = 0`。

## 各层职责

| 层 | 工具 | 作用 | 只在什么模式 |
|------|------|------|:--:|
| 市况判 | 深证通道状态 | 分叉到单边/震荡逻辑 | 始终 |
| L1 月九转 | 月低9/高9 | 大周期底部/顶部区域 | 始终 |
| L2 周九转 | 周低9/高9 | 中期节奏确认 | 始终 |
| L3 日线 | 深证通道+五共振 | 操作主信号 | 始终 |
| L4 结构 | MACD顶钝/底钝/结构 | 执行时机 | 始终 |
| L0 连错 | 趋势信号失败计数 | 震荡末期预判 | **仅震荡** |
