# 操作层 · 数据管线说明

## 脚本

| 脚本 | 功能 | 输出 |
|------|------|------|
| `scripts/fetch_data.py` | 拉取六大指数日线 + 周月线重采样 + MA20/60通道 | `data/daily_ma_channels.csv` 等 |
| `scripts/structure_engine.py` | MACD(4,30,4) 钝化→结构形成→状态机 | 日线结构信号（被其他脚本 import 使用） |
| `scripts/market_classifier.py` | 徐式市况分类 v2（③a/③b/③c 含结构集成） | `data/market_condition_xu_v2.csv` |
| `scripts/sequence_engine.py` | 九转序列 高9/低9（日/周/月 + 钝化+9增强） | `data/turn_sequence_events.csv` |
| `scripts/unimportant_detector.py` | "不重要"检测（不改变操作结果 = 不重要） | `data/unimportant_daily.csv` |
| `scripts/failure_tracker.py` | 趋势连错追踪 v2（六指数 + 结构过滤假阳性） | `data/failure_tracker_v2.csv` + summary |

## 指数

上证/深成指/创业板/科创50/沪深300/中证500（六指数）

## 参数选择

- **MA20/MA60**：双通道默认。MA20短期趋势，MA60长期趋势。
- **MACD(4,30,4)**：结构判定专用参数，lookback=40天+级别≥2过滤。
- **连错确认窗口**：20天，价格变动>2%确认方向。

## ═══ 交叉分析原则（防单指标陷阱） ═══

> **核心**：趋势/结构/序列三者不可独立使用。单看任何一个都会掉进"假信号"陷阱。必须三层交叉验证，按优先级过滤。

### 优先级链（徐小明原文）

```
趋势 > 结构 > 序列
```

"这个排序的意义在于如果这三者之间起冲突的时候，谁要服从谁。"

### 三层过滤决策

```
① 趋势层：趋势方向 vs 序列方向
   - 同向 → 继续往下走
   - 反向 → 序列降权。"序列的缺点就是丢趋势，一个趋势性的行情，序列几乎都是不对的"
   - 回测：30%的序列失效中，60%发生在逆趋势场景

② 结构层：有无钝化/结构形成
   - 有结构共振 → 增强信号可信度
   - 无结构 → 观望。"上升趋势里的序列高9最好能够配合结构"
   - 钝化+9 → 调整级别更大、更可信

③ 序列层：序列本身的有效性
   - 8-10 均为有效值
   - 月线 > 周线 > 日线（周期评分）
   - 多指数共振 ≥3 → 更可靠
```

### 典型场景速查

| 趋势 | 结构 | 序列 | 可信度 | 操作 |
|------|------|------|--------|------|
| 震荡 | 有钝化 | 高9 | **高** | 准备减仓，等结构形成 |
| 震荡 | 无 | 高9/低9 | 中 | 观察，"随缘"，不作为操作依据 |
| 上升趋势 | 无 | 高9 | **低** | 序列让位趋势，"序列属于小聪明，趋势是大智慧" |
| 上升趋势 | 顶部钝化 | 高9 | 中高 | 有调整需求，但级别取决于结构 |
| 下降趋势 | 无 | 低9 | **低** | 序列让位趋势 |
| 下降趋势 | 底部结构 | 低9 | 中 | 可信反弹，但仅限反弹不是反转 |

## 结构信号集成

### 连错假阳性过滤（v2新增）
```
规则：连错≥4 且 连错期间 顶钝化次数/底钝化次数 > 3:1
     且 80%以上结构形成为单向 → 标记为疑似假阳性
原理：真转折时震荡来回拉锯，顶底结构信号均衡；
     单边结构积累说明市场实际有方向，MA20穿越是噪音。
未来增强：多指数同时触发假阳性 → 可信度加权。
```

### 市况子分类（v2新增）
```
③a 震荡·有信号 = 震荡 + 至少1个指数处于钝化/结构形成中 → 可以等操作
③b 震荡·无信号 = 震荡 + 所有指数均无结构 → 多看少动/随缘
③c 震荡·收敛末期 = 震荡≥90天 + 多指数通道收敛 → 即将切换
```

结果：震荡市 85% 无信号（多看少动），15% 有结构积累可操作。

### "不重要"检测（徐小明核心工具）

**原文定义**（2026-06-04）：
> "不重要 = 不改变操作结果。操作结果由两件事决定：
>  ① 趋势是否被跌穿/突破  ② 钝化是否消失/结构是否形成
>  过程是怎样波动的都不能导致结果产生，这些过程就都不重要。"

**检测规则**：六指数任一触发以下即算"改变操作结果"：
- 趋势突破/破位（收盘上穿MA20且MA20>MA60 / 下穿MA60且MA20<MA60）
- 结构形成（顶部/底部结构）
- 钝化消失（纠错信号）

**实测结果**：
- 改变操作结果: 709天 (38.9%) — 需要决策
- **不改变结果: 1114天 (61.1%) — 不重要**
- 最长连续不重要: 42天
- 震荡·无信号中 76% 不重要，震荡·有信号仅 32%

**方法定位**：这是徐小明"化繁为简"的核心工具。不输出"该做什么"，而是输出"今天是否需要做决策"。61%的交易日可被过滤。

### 九转序列（v3新增）
- **定义**：收盘价 vs 4根前收盘价，连续9次同向 = 高9/低9
- **覆盖**：日线（年均4-5段/指数）+ 周线 + 月线（共2-3段/指数）
- **增强**：钝化+9 = 同周期钝化+序列9重叠（日线仅2-7次/指数/8年）
- **成功率**：整体~70%，逆趋势场景降60-65%
- **关键验证**：2023-12~2024-02 四指数月线低9 → 徐小明大底判断完全吻合

## 已知坑

- 连错只在震荡市积累，单边市不触发（2024.05-09是70%下降趋势，无连错正常）
- 结构信号需级别过滤（divergence_level≥2），否则噪声过大（62→13次/7.5年）
- numpy `np.select` 返回固定长度 dtype，长标签需加 `.astype(object)`
- 收敛末期（③c）条件严格（≥90天+通道收敛），2019-2026仅0天——待观察是否需要调参
- 序列不能独立使用——必须与趋势、结构交叉验证（见上方交叉分析原则）
