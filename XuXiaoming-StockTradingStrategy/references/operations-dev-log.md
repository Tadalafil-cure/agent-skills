# 操作层开发要点 · 本次会话实录

## 数据管线建设顺序

按缺口优先级逐一构建，每个模块先验证再继续：

1. **fetch_data.py** — 基础数据：六指数日线 + MA20/MA60通道 + 周月重采样
2. **structure_engine.py** — 结构判定：MACD(4,30,4) 钝化→结构形成→生命周期状态机
3. **market_classifier.py** — 市况分类：多数投票，按徐小明自己的标签（单边上升/下跌/震荡），不用外部阶段标签
4. **failure_tracker.py v2** — 连错追踪升级：六指数全量 + 结构引擎交叉验证
5. **market_classifier.py v2** — 市况分类升级：接入结构引擎，实现③a/③b/③c 震荡子分类

## 九转序列（2026-07-11）

### 序列引擎构建
- **定义**：TD Sequential，收盘价 vs 4根前。连续9次同向=高9/低9。相等=断裂
- **覆盖**：日/周/月三周期六指数 + 日线 钝化+9 增强
- **日报信号太多**：原始 535 次共振（连续多天重复统计同一序列段）
- **去重策略**：按序列段去重——找 count≥8 的起始点，同段内只取 count=9 那天，降至 124 次共振
- **月线验证**：2023-12~2024-02 四指数月线低9 → 与徐小明"四个指数月线序列"大底判断完全吻合
- **钝化+9**：同周期钝化状态+序列9重叠，日线仅 2-7 次/指数/8年，极稀少增强信号

### 连错假阳性过滤
- **过滤条件**：连错≥4 且 顶钝化/底钝化比 > 3:1 且 结构形成80%+单向 → 假阳性
- **原理**：真震荡积累中顶底结构信号均衡（来回拉锯），单边结构说明市场有方向
- **实测**：3次假阳性正确识别（2021-07深证、2025-07创业板、2025-10上证，钝化比 4:1~20:1）
- **影响**：≥4连错事件从 10 次过滤至 7 次真信号
- **未来增强**：多指数同时触发假阳性 → 可信度加权（用户建议）

### 市况子分类
- **③a 震荡·有信号**：至少1个指数处于钝化/结构形成中（142天/15%震荡期）
- **③b 震荡·无信号**：所有指数均无结构（794天/85%震荡期）→ "多看少动"
- **③c 震荡·收敛末期**：震荡≥90天 + 通道收敛（0天）→ 条件可能过严
- **实现**：market_classifier.py import structure_engine，六指数每日结构状态聚合

### "不重要"检测（2026-07-11，替代原三无）

**原文定义**: "不重要 = 不改变操作结果。操作结果由两件事决定：①趋势是否被跌穿/突破 ②钝化是否消失/结构是否形成。过程是怎样波动的都不能导致结果产生，这些过程就都不重要。"

- **为什么替代三无**："三无行情"仅2019年提及，后被"过程中的交易日不重要"替代（2024-2026高频出现，2025年27次/2026半年19次）
- **检测规则**：六指数任一触发趋势突破/破位 + 结构形成 + 钝化消失 = 改变操作结果
- **实测**：709天(38.9%)改变结果 ← 需要决策；1114天(61.1%)不改变 ← 不重要
- **方法定位**：徐小明"化繁为简"核心工具。不输出"该做什么"，输出"今天是否需要做决策"
- **关键数据**：最长连续不重要42天，震荡·无信号中76%不重要，震荡·有信号仅32%

### 规则引擎 + 全管线全检（2026-07-11）

**管线全检结论**：12/12 原文对照通过，7引擎零逻辑违规。

**关键修正**（用户反馈）：
- **"不重要"逻辑顺序**："不重要"是综合裁决后的结论，不是前置过滤器。正确流程：先跑完所有信号（趋势/结构/序列/连错/协调）→ 再判断是否改变操作结果。用户："是不是逻辑上面应该是先把各个这些结构趋势结构，包括序列都已经列好了，然后再来做这些后面的判断"
- **输出格式**：用户需要先看到大环境（什么阶段、什么市况、什么趋势），再看到具体信号和建议。简报四层级：①大环境→②信号→③判断→④关注
- **9/24不强求**：政策发动速度太快，徐小明自己也未预测到。引擎只需在前后有足够信号反应即可。

**规则引擎操作分布**：观望61%/纠错14%/关注突破7%/满仓5%/结构操作4%/减仓4%/左侧试仓4%/空仓1%。年均~10次实质操作，与时期三4-8次/年吻合。

**关键日期验证**：
- 2021-02-18 (3731大顶): 重要→减仓防守 ✅
- 2022-04-27 (2863底): 重要→左侧试仓(严重分化降权) ✅
- 2024-09-24 (突破): 重要→结构操作(趋势尚未跟上但结构先触发) ✅
- 2024-10-08 (政策顶): 不重要→观望(无结构形成，序列不足以触发) ✅
- **序列成功率**：整体~70%，10天窗口达标率60%，20天70%
- **失败原因**：60%发生在逆趋势场景。"序列的缺点就是丢趋势"——徐小明原文
  - 高9 + 上升趋势 → 35%失败率（趋势碾压序列）
  - 低9 + 下降趋势 → 38%失败率
- **交叉分析原则（防单指标陷阱）**：
  - 趋势方向 vs 序列方向 → 反向则序列降权
  - 结构钝化/形成 vs 序列信号 → 共振则增强
  - 三层过滤：趋势→结构→序列，按优先级逐层否决
  - 典型：上升趋势+高9+无结构 → 序列让位；震荡+钝化+9 → 高可信
- **原文验证**：
  - 月线低9四指数共振 2023-12~2024-02 → 徐小明大底判断完全吻合
  - "序列属于小聪明，趋势是大智慧"（2020原文）

## 关键坑

### 通道算法
- ❌ 旧管线用"滚动高低点通道"(rolling high-low)
- ✅ 徐小明用的是 MA20/MA60 均线通道
- 这个错了后面全错——H1趋势突破/破位直接依赖它

### 结构判定
- ✅ MACD(4,30,4) for 结构判定（不是默认12,26,9）
- lookback=40天（不是20天），否则信号太多（62次→13次）
- 需加 divergence_level 过滤：≥2（中级）才算有效结构

### 趋势信号定义
- ❌ 裸 MA20 穿越 → 227次信号，噪音太大，最大连错仅3
- ✅ MA20/MA60 双通道：突破=站上MA20且MA20>MA60，破位=跌破MA60且MA20<MA60，20天确认
- 正确后：52次信号，最大连错8次

### 连错七次
- 徐小明原文："我判断8、9、10这三次有一次走单边市"——7是统计观察，非精确触发
- 实测7.5年仅1次≥7，实用阈值：≥5关注，≥7预警
- 连错信号只告诉你"要变了"，不告诉方向（2019Q4→上涨，2022Q1→下跌）

### 指数选择
- 初始：上证/深成指/创业板/上证50/沪深300/中证500
- 用户修正：去掉上证50（和沪深300高度重叠），加入科创50（代表成长风格）
- 最终六个：上证/深成指/创业板/科创50/沪深300/中证500

### 市况分类
- 用户强调：用徐小明自己的标签体系（单边上升/下跌/震荡），外部6阶段只是参考坐标系
- 徐小明操作层面有3种状态（哲学上只承认2种属性）
- 震荡占全时期51.3%——验证了"大部分时间是不重要的时期"
