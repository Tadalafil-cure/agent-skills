# 盘后全流程执行清单

每次生成操作策略报告时，按此清单顺序执行，缺哪步停哪步。

```
-1  日期确认       date "+%Y-%m-%d %A"
 0  市场状态       market_status() → 盘前/盘中/盘后
 1  数据管线       build.py（增量，不 rm 已有数据）
 2  分钟线         fetch.py --mode minute（60→120, 30→90）
 3  管线产出       process.py（验证 7 文件）
 4  交叉验证       日线|结构|序列|裁决|分钟线四指数|五指数投票明细
 5  主Agent统稿    一~八节 ⛔不碰博客·不拆子Agent
 6  **校验Agent**  逐条对照关键规则审查一~八节定稿。首次不通过→打回主Agent修正→重审。二次仍不通过→不再打回，列出未通过项直接报告用户，继续向下推进。
 7  博客Agent      A20隔离·子Agent返摘要·主Agent不接触原文
 8  追加比对       仅第九节·不修改一~八节
 9  清理           旧报告/子Agent散落文件删除·reports/只保留最终版
```

## 关键规则

- **不拆子 Agent 做七层递进**：主 Agent 自己读数据、自己推演、自己写。拆成多个子 Agent 产生矛盾结论（diverge 含义不一致、裁决判法不一致），协调成本 > 分析收益。
- **博客后置**：主 Agent 写完一~八节后，才派独立子 Agent 读博客。A20 三层隔离——主 Agent 不接触博文原文。
- **增量更新**：已有 daily_raw.csv 直接复用，build.py 的时效检测会自动补缺失交易日。不要 rm 清空重拉。
- **共振投票表必写**：报告第一节必须包含五指数共振投票明细表（上证/深证/创业板/沪深300/中证500各自 regime + 方向 + 票数），不能只写"混合"两个字。
- **钝化回测结论必嵌入**：跨市场交叉分析节必须引用 `references/divergence-backtest-v4611.md` 的回测数据，标注当前分钟线顶/底钝化的可靠性等级。分钟线底钝化单用不可靠（假信号46-100%），必须结合日线低9/CHOP解读。
- **CHOP 分级铁律**：<38.2=clear，38-62=fuzzy，>61.8=chaotic。不能写"CHOP<50=clear"——这是错的。
- **A19 分列**：顶/底结构按市况（趋势中/震荡中）分开统计，论证只用趋势中子数。
- **diverge=-1 = 无背离**：分钟线旧版 diverge 列含义。v4.6.11 起 bd/td 分离，bd=1=底钝化，td=1=顶钝化。
- 🔴 **日期校验必须用 `date` 命令**：禁止凭 session 上下文中的"会话开始日期"推断当前日期。会话可能跨越多天（如漫画项目耗时数周），上下文日期 ≠ 实际日期。每次分析前必须 `date "+%Y-%m-%d %A"`。
- 🔴 **A20 博客隔离——一~八节零博客引用**：写完一~八节前，报告中不得出现"徐小明说""徐小明原文""徐小明认为"、不得出现博客中的具体点位（3980/3985）、不得出现博客中的金句（"来啥接啥""倒数第二笔"）。违反一次=整篇报告作废，重写一~八节。
- 🔴 **A20 执行隔离——引擎数据与博客分两轮调用**：`blog_monitor.py` 不得与引擎数据查询（裁决/序列/结构/分钟线/NHNL）出现在同一个 `terminal` 调用或 `&&` 链条中。第一轮只拉引擎数据→写完一~八节→第二轮独立运行 `blog_monitor.py`→追加第九节。两个调用之间物理隔离，中间不存在任何博客内容进入上下文的可能性。已犯3次（7/10+7/14+7/16），根因：博客和引擎数据同轮拉取→博客渗透分析段。
- 🔴 **H17b 上行来路底结构抑制**：震荡来路=上行时，底部结构不触发试探/持股——仅关注。逻辑：上行趋势顶结构已减仓→仓位已低→震荡中底结构是反弹不是反转。裁决引擎 v4.6.13 已实现：`osc_origin==上行` + bs=1 → `观望` + 标注"来路上行+底结构→关注(不触发)"。
- 🔴 **M1 规则独立——CHOP 回落 ≠ M1 解锁**（7/16 教训）：M1 解锁条件与趋势可信度走向相反。CHOP 越高（>61.8 chaotic）分钟线底结构越可靠，CHOP 越低（fuzzy 区）噪音率 29% 反而需要禁用。不能把"CHOP 在改善"的逻辑链延伸到 M1——两条线各自独立判断，互不传递。
- 🔴 **A18 分钟线强制报告**：四指数分钟线状态必须出现在报告二/三节各自的推演中 + 四节跨市场交叉分析汇总。即使四个指数全为"无信号"，也必须逐指数标注"7/16 无信号 + M1 禁用/不禁用状态"。引擎存在即责任——跑了不写 = 有工具不用。
- 🔴 **A18e 钝化数据必嵌入**：裁决不依赖钝化，但分析必须包含当前钝化状态。报告跨市场交叉分析节必须嵌入 v4.6.11 钝化回测结论（日线转化88-95%、分钟线顶钝化89-98%可靠、底钝化单用不可靠需结合日线低9/CHOP）。即使当前无钝化（bd=0），也须标注"无钝化"并引用回测基线。
- 🔴 **A19b 统计分开 ≠ 思维分开**：A19 要求表格分市况列示（趋势中/震荡中），但仅表格分开不够——后续所有引用结构总数的论证必须锚定切分后的子数。禁止说"27次顶结构 vs 3次底结构"，应说"趋势中20顶 vs 3底 = 6.7:1，震荡中另7顶为区间标记不参与压制论证"。
- 🔴 **A23 五指数共振成员固定**：五指数 = 上证+深证+创业板+沪深300+中证500。科创50明确排除——仅参与科技板双创共振。报告中五指数投票表不得出现科创50。
- 🔴 **A24 偏多标签须溯源**：裁决"观望(偏多)"的偏多后缀必须标注具体来源（低9/来路上行/CHOP收敛等），区分"单个指数关注偏置"vs"共振背书的方向判断"。全震荡+混合时，偏多只是个体意见，不代表多指数合力。
- 🔴 **A24b 来路独立计算——禁止跨指数同步**（7/16 教训）：每个指数的来路由自身 regime 历史独立判定，互不同步。引擎 `osc_origin` 字段逐指数计算，不存在"主板来路上行→科技侧同步"。常见错误：看到科创50有偏多而创业板没有→归因于"主板同步"或引擎 bug。正确答案：创业板自身震荡前无≥7天上行趋势段→来路不明→不附偏多；科创50自身有→来路上行→附偏多。各算各的，无传递。
- **A24c osc_origin 引擎覆盖范围**：引擎仅导出 `osc_origin_sz` 和 `osc_origin_sh` 两列。创业板和科创50的来路需从 `reason_cyb`/`reason_kc` 字段提取（如"来路不明"、"来路上行→偏多"）。报告四指数来路表中创业/科创行数据来源标注为 reason 字段，不虚构 osc_origin 值。
- 🔴 **A28 禁止轻量输出**：无论用户措辞多轻量（"看一下""分析一下""快速看看"），触发本 skill 一律走完整九节模板。不存在快捷模式。
- 🔴 **文件名规范**：`reports/YYYY-MM-DD_主题_YYYYMMDD_HHMMSS.md`。日期=分析基准日，HHMMSS=生成时间。
