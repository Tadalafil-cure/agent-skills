# 盘后全流程执行清单

每次生成操作策略报告时，按此清单顺序执行，缺哪步停哪步。

```
-1  日期确认       date "+%Y-%m-%d %A"
 0  市场状态       market_status() → 盘前/盘中/盘后
 1  数据管线       build.py（增量，不 rm 已有数据）
 2  分钟线         fetch.py --mode minute（60→120, 30→90）
 3  管线产出       process.py（验证 7 文件：structure/sequence/verdict + 4×minute_structure_v2）
 4  交叉验证       日线✅ | 结构✅ | 序列✅ | 裁决✅ | 分钟线四指数✅ | 五指数投票明细✅
 5  主Agent统稿    一~八节 ⛔不碰博客·不拆子Agent
 6  博客Agent      A20隔离·子Agent返摘要·主Agent不接触原文
 7  追加比对       仅第九节·不修改一~八节
 8  清理           旧报告/子Agent散落文件删除·reports/只保留最终版
```

## 关键规则

- **不拆子 Agent 做七层递进**：主 Agent 自己读数据、自己推演、自己写。拆成多个子 Agent 产生矛盾结论（diverge 含义不一致、裁决判法不一致），协调成本 > 分析收益。
- **博客后置**：主 Agent 写完一~八节后，才派独立子 Agent 读博客。A20 三层隔离——主 Agent 不接触博文原文。
- **增量更新**：已有 daily_raw.csv 直接复用，build.py 的时效检测会自动补缺失交易日。不要 rm 清空重拉。
- **共振投票表必写**：报告第一节必须包含五指数共振投票明细表（上证/深证/创业板/沪深300/中证500各自 regime + 方向 + 票数），不能只写"混合"两个字。
- **钝化回测结论必嵌入**：跨市场交叉分析节必须引用 `references/divergence-backtest-v4611.md` 的回测数据，标注当前分钟线顶/底钝化的可靠性等级。分钟线底钝化单用不可靠（假信号46-100%），必须结合日线低9/CHOP解读。
- **CHOP 分级铁律**：<38.2=clear，38-62=fuzzy，>61.8=chaotic。不能写"CHOP<50=clear"——这是错的。
- **A19 分列**：顶/底结构按市况（趋势中/震荡中）分开统计，论证只用趋势中子数。
- **diverge=-1 = 无背离**：分钟线旧版 diverge 列含义。v4.6.11 起 bd/td 分离，bd=1=底钝化，td=1=顶钝化。
