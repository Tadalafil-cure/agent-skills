## 默认工具

规则层 8 组规则是框架无关的。以下 7 个指标是**默认实现**，用户可任意替换：

| 规则接口 | 默认工具 | 参数 | 可替换为 |
|---------|---------|------|---------|
| 趋势方向 | 三合一投票（ER30 + ADX14 + 动量20） | 250天65%分位，非对称滞回 | MA通道、Donchian、自定义分类器 |
| 趋势可信度 | CHOP(14) | 3级：clear(<38.2) / fuzzy(38-62) / chaotic(>61.8) | ADX强度、ATR波动率 |
| 结构信号（日线） | MACD(4,30,4) 钝化→结构形成 | 日线 | 自定义MACD参数、RSI背离 |
| 结构修边（分钟线） | MACD(4,30,4) 三周期共振 | 60/90/120 分钟 | 自定义分钟周期组合 |
| 序列信号 | TD Sequential 九转 | 日/周/月 | 自定义序列算法 |
| 多指数共振 | 五指数三合一方向统计 | 上证/深证/创业板/沪深300/中证500（科创50仅参与双创共振） | 任意指数组合 |
| **个股与大盘分化** | **NH/NL双公式（CNHL + HL_Ratio）** | **N=60主/N=20辅** | **涨跌家数、自定义广度指标** |

## 接入自己的工具

只要你的工具能提供规则层要求的输出格式，裁决引擎就能自动运行：

1. **趋势方向** → `{date, regime(上行/偏多/震荡/偏空/下行), bullish}`
2. **趋势可信度** → `{date, chop_value, chop_level(clear/fuzzy/chaotic)}`
3. **结构信号** → `{date, bottom_structure, top_structure}`
4. **广度信号** → `{date, nh, nl, cnhl, hl_ratio, stage}`（通过 `data_layer/breadth.py` 或自定义）
5. 调用 `scripts/verdict_v7.py` 的 `verdict_single()` 即可

规则优先级（铁律）不会变：**趋势为王 > 结构修边 > 序列辅助**。

## 裁决出口

| 裁决 | 含义 | 后续 |
|------|------|------|
| 持股 | 满仓持有，趋势不破不动 | 顶部结构→持股(警戒)；二次→减仓 |
| 试探 | 底部信号出现，小仓位试多 | 月周低9窗口内可升档为持股 |
| 观望 | 等待信号，多看少动 | 等趋势突破或底部结构 |
| 空仓 | 清仓防守 | 等趋势突破或底部结构+BS通过 |
| 减仓 | 降低仓位，防守为主 | 等新的底部信号 |

## 数据管线

```bash
# 一键构建：拉取 akshare → 处理 → 输出全部 CSV（首次自动装依赖）
python data_layer/build.py

# 裁决引擎
python scripts/verdict_v7.py

# 广度引擎（v4.5.2 新增）
python data_layer/breadth.py    # 输出结构化 JSON：stage/ratio/cnhl/divergence
```

产出：
- `data/verdict_v7.csv`（40+列，四指数独立裁决 + 两市场裁决 + 五指数共振 + 双创共振）
- `data/breadth_daily.csv`（NH/NL/CNHL/HL_Ratio + 阶段判定）

## 诚实边界

- 纯指数级技术分析，不涉及选股、基本面、板块轮动
- 个股分化分析基于 NH/NL 新高新低家数（广度指标），不等同于选股
- MACD 参数、钝化阈值、序列分数未在公开文章中披露
- 2026 年仅 H1
- 模型在震荡期偏保守——方向一致的案例中，强度差异占多数（方向矛盾 <5%）
- NH/NL 数据仅 500 天回溯（akshare 限制），回测结论不能外推到全周期
