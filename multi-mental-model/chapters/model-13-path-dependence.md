# 模型13：路径依赖

**核心检验问题**：护城河在生态/合规/惯性，还是只靠产品？

**运用要点**：
1. 护城河远不止产品本身——生态锁定、合规认证、客户采购惯性、供应链认证才是真正的壁垒
2. 大型客户一旦接入某家供应商，迁移意味着安全、责任、运维/产线全部重做——这是典型的路径依赖锁定
3. 「换供应商不过是改个合同」的说法只在理论层面成立——现实中，产线认证可长达数月甚至数年
4. 产品差异化程度低 ≠ 无护城河——如果客户切换成本高，同质化产品仍可形成强锁定
5. 判断护城河需要完整同业对比——缺乏时标注不确定性

**诊断清单**：
- [ ] 论文是否仅从产品同质化直接推断「无护城河」？
- [ ] 客户的切换成本是什么？涉及哪些环节（认证/合规/产线/配方）？
- [ ] 龙头企业是否有已建立的供应链惯性？
- [ ] 「P_同业分析报告缺失」本身就是护城河判断的盲区
