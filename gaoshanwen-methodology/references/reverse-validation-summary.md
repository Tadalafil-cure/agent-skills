## 系统性发现

### 铁律③（结论必含证伪条件）——最高频弱项

32 篇中约 40% 出现铁律③的轻度或中度违规。这是**高善文本人的写作特征**而非技能缺陷——他倾向于在文章结尾集中给出总证伪条件，而非逐段为每个中间结论标注。A 模式深度研报和默认模式策略会演讲均存在此模式。

**实用建议**：使用 skill 生成内容时，可主动在中间结论后追加"若【X数据】出现【Y方向】变化，则此判断需要重新评估"句式，弥补原文风格上的缺口。

### B模式退化分支——需显式指令

悼文/纯个人随笔（001送别父亲、007悼词李勇）触发 B 模式退化分支时，若子 agent 未收到显式退化指令，会按 B 模式默认参数（1-2模型、锋利25-40%）强行注入经济学框架。**给子 agent 的 prompt 中必须显式声明"方法论=0、锋利=0、自嘲=0"。** 直接使用 skill 时（非子 agent 场景），用户会自然说"帮我写一段悼文"，退化规则可正常触发。

### 表达参数微调（2.8规则）——验证通过

083 企业资产负债率一文锋利度 10-15%（低于默认模式 20-30% 基准），但在 2.8 规则中触发"确定性结构性判断 + 数据密集型分析"双偏移条件，落在可接受范围内。验证了 2.8 规则的实用性和解释力。

### A 模式表达参数补充

048 盈虚之有时暴露的"概念化过度"问题，根源在于 A 模式下每个论断需**具体数据支撑**而非抽象概念推演。这个约束已在 v4 表达层参数表中明确标注（"数据叙事密度"行），需要在实际使用中强制执行。
