## 1. support_resistance.py（支撑/压力位 + 转换判定）

**状态：** 零，待方法论

**功能需求：**

### 1.1 支撑/压力位识别
| 功能 | 说明 |
|:--|:--|
| Pivot high/low 检测 | 局部极值点识别（窗口可配，默认左右各 N 根 K 线） |
| 水平位聚合 | 相近价位（±x%）的多个高低点合并为一个支撑/压力区 |
| 触点计数 | K 线触及该价位的次数，次数越多级别越高 |
| 多时间窗 | 近 20/60/120/250 日分别识别 |

### 1.2 转换判定规则
| 功能 | 说明 |
|:--|:--|
| 突破确认 | 收盘价站稳原压力位上方 ≥ N 日 + 成交量放大 ≥ M%（可配） |
| 回踩确认 | 突破后回调最低点 ≤ 原压力位 ± x%，随后再次站上 |
| 转换标注 | 旧阻力 → 新支撑（带确认状态：待确认/已确认/失败） |

**输出格式：** 各时间窗的支撑位列表 + 压力位列表 + 转换记录（全确定性字符串，不做强弱解读）

**参考方法论来源：** 待补充（需找 Pivot Points / Murrey Math / 订单流相关的公开方法）

## 2. multi_tf.py 共振强度（扩写现有脚本）

**状态：** 四档定性分类，无量化分数，**待方法论**

**当前输出：**

**缺陷：** 周/日/60min 三周期全上升和两上升一震荡，输出完全相同，强度差异被抹平。

**需求：**
| 功能 | 说明 |
|:--|:--|

| 强度分档 | 基于加权分输出 `resonance_score` 和 `alignment_detail` |
| 周期明细 | 每个周期的趋势方向 + 强度（如 MA 多头排列程度） |

**输出新增字段：**
```json
{
  "resonance_score": 4.5,       // 连续值 [-6, +6]
  "alignment": "一致看多",       // 保留原定性
  "alignment_detail": {          // 新增明细
    "weekly": {"trend": "上升", "ma_score": 0.8},
    "daily": {"trend": "上升", "ma_score": 0.9},
    "hourly": {"trend": "震荡", "ma_score": 0.1}
  }
}
```
